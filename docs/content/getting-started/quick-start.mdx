# Quick Start

This guide walks you through the full MorphoCLIP pipeline: downloading data, extracting features, and training the model.

Tasks are defined under `[tool.poe.tasks]` in `pyproject.toml` and run through [poe](https://poethepoet.natn.io/)
via `uv run poe <task>`. Each task is a thin wrapper around a `python scripts/...` call, so you can always run
the underlying script directly if you prefer.

## Command Reference

| Task                            | Command                        | Underlying script                                                   |
| ------------------------------- | ------------------------------ | ------------------------------------------------------------------- |
| Run tests                       | `uv run poe test`              | `pytest tests/ -v`                                                  |
| Fetch dataset from S3           | `uv run poe fetch-dataset`     | `python scripts/data/fetch_dataset.py`                              |
| Extract DINOv3 features         | `uv run poe extract-features`  | `python scripts/features/extract_features.py`                       |
| Download pre-extracted features | `uv run poe download-features` | `python scripts/features/download_features.py`                      |
| Pre-compute text embeddings     | `uv run poe precompute-text`   | `python scripts/text/precompute_text_embeddings.py`                 |
| Train model                     | `uv run poe train`             | `python scripts/training/train.py --config configs/train/base.yaml` |
| Evaluate model                  | `uv run poe eval`              | `python scripts/training/eval.py --checkpoint <path>`               |
| Run inference                   | `uv run poe infer`             | `python scripts/training/infer.py --checkpoint <path>`              |

## Training Pipeline

```bash
# 1. Download pre-extracted DINOv3 features from Hugging Face
uv run poe download-features

# 2. Pre-compute text embeddings (once)
uv run poe precompute-text

# 3. Train MorphoCLIP (default config configs/train/base.yaml)
uv run poe train

# 4. Evaluate a trained checkpoint
uv run python scripts/training/eval.py --checkpoint output/morphoclip_runs/default/checkpoints/best.pt

# 5. Run inference (top-k matching, embedding export, or profile export)
uv run python scripts/training/infer.py --checkpoint output/morphoclip_runs/default/checkpoints/best.pt
uv run python scripts/training/infer.py --checkpoint output/morphoclip_runs/default/checkpoints/best.pt --mode embed
uv run python scripts/training/infer.py --checkpoint output/morphoclip_runs/default/checkpoints/best.pt --mode profile
```

## Training Configuration

The default config is at `configs/train/base.yaml`. Key options:

- `loss_type`: `"infonce"` (standard contrastive) or `"cwcl"` (soft-label contrastive)
- `use_cwa`: `true` to enable Cross-Well Alignment batch correction (disabled by default)
- Outputs saved to `output/morphoclip_runs/<run_name>/`

Alternative configs:

- `configs/train/mean_pool.yaml` — uses mean pooling instead of CrossChannelFormer
- `configs/train/ddp.yaml` — multi-GPU distributed training

## Multi-GPU Training

Replace `poe train` with a direct `torchrun` invocation and add `--distributed`:

```bash
# MorphoCLIP on 4 GPUs
uv run torchrun --nproc_per_node=4 scripts/training/train.py --config configs/train/base.yaml --distributed

# CellCLIP on 4 GPUs
uv run torchrun --nproc_per_node=4 scripts/cellclip/train_cellclip.py --config configs/cellclip/cellclip_1024dim.yaml --distributed
```

The batch size in the config is **per-GPU**, so effective batch = `batch_size x num_gpus`.

## TensorBoard Monitoring

Training logs metrics to TensorBoard automatically (tensorboard ships with the project deps):

```bash
uv run poe tensorboard                                          # MorphoCLIP (output/morphoclip_runs)
uv run tensorboard --logdir output/train_runs/ --port 6006      # CellCLIP
```

If running on a remote machine, forward the port over SSH: `ssh -L 6006:localhost:6006 user@remote-host`
